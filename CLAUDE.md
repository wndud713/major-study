# PDF → HTML 스터디 파일 변환 지침

## 파일 구조
- `shell_template_v2.html` = 반드시 따라야 할 디자인 표준
- PDF = 콘텐츠 원본

## 절대 규칙
- 캐러셀 엔진 JS 절대 수정 금지
- 디자인 시스템 `shell_template_v2.html` 그대로 준수
- 새해부학 용어 사용 (비복근→장딴지근 등)
- 결과 HTML은 같은 폴더에 저장
- 완료 후 변경사항만 요약 보고

## 텍스트 추출
- `pdftotext -layout "파일.pdf" -` 로 슬라이드 텍스트 추출
- 스캔본(텍스트 없음)이면 슬라이드 이미지를 Claude Vision으로 직접 분석

## 이미지 크롭 기준
- 크롭 대상: 도식·그림·표 영역만
- 슬라이드 배경·제목·텍스트 불릿은 제외
- 이미지/도식이 없는 슬라이드는 크롭 건너뜀
- `pdfimages`로 임베디드 이미지 먼저 추출
  → 없을 경우에만 `pdftoppm`으로 래스터화 후 크롭

## 콘텐츠 구조 원칙
- 카드: 핵심 3~5개 청킹 (훑어보기)
- 디테일 패널: PDF 내용 빠짐없이 전부 수록 (깊이 보기)
- 도식·표·구조화 우선
- 용어는 맥락·어원과 함께 설명
